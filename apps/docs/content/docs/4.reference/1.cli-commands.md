# CLI Commands

Complete reference for Code Please CLI commands.

## code

Main CLI tool for formatting and LSP operations.

### code format

Format a file using the appropriate formatter.

```bash
code format <file>
code format --stdin
```

**Arguments:**
- `<file>` - Path to file to format

**Options:**
- `--stdin` - Read file info from stdin (hook mode)

**Examples:**

```bash
# Format a TypeScript file
code format src/index.ts

# Format via hook (stdin mode)
echo '{"tool_input":{"file_path":"src/index.ts"}}' | code format --stdin
```

### code lsp

Get LSP diagnostics for a file.

```bash
code lsp <file>
code lsp --stdin
```

**Arguments:**
- `<file>` - Path to file to check

**Options:**
- `--stdin` - Read file info from stdin (hook mode)

**Examples:**

```bash
# Check a TypeScript file
code lsp src/index.ts

# Check via hook (stdin mode)
echo '{"tool_input":{"file_path":"src/index.ts"}}' | code lsp --stdin
```

### code lsp-server

Manage LSP servers.

```bash
code lsp-server list
code lsp-server info <server>
```

**Subcommands:**
- `list` - List available LSP servers
- `info <server>` - Show details for a server

### code version

Show version information.

```bash
code version
```

### code help

Show help information.

```bash
code help
code help <command>
```

## dora

MCP server for IDE integration.

### dora serve

Start the MCP server.

```bash
dora serve
```

The server runs on stdio for MCP client integration.

### dora version

Show version information.

```bash
dora version
```

### dora help

Show help information.

```bash
dora help
```

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success |
| 1 | Error occurred |

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `CODE_PROJECT_PATH` | Project root for code | cwd |
| `CLAUDE_PROJECT_DIR` | Project dir from Claude Code | - |
| `DORA_PROJECT_PATH` | Project root for dora | cwd |
| `DORA_TIMEOUT` | Request timeout (ms) | 30000 |

## Stdin Format

When using `--stdin` mode, input is JSON:

```json
{
  "tool_name": "Write",
  "tool_input": {
    "file_path": "/path/to/file.ts",
    "content": "..."
  }
}
```
