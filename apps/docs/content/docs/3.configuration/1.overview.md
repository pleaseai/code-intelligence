# Configuration Overview

Code Please uses a unified configuration file to customize formatter and LSP behavior.

## Configuration File

Create `.please/config.yml` or `.please/config.json` in your project root:

::code-group
```yaml [.please/config.yml]
language: en

ignore_patterns:
  - node_modules
  - dist
  - .git

formatter:
  biome:
    extensions: [.ts, .tsx, .js, .jsx]
  prettier:
    disabled: true

lsp:
  typescript:
    enabled: true
  oxlint:
    enabled: false
```

```json [.please/config.json]
{
  "language": "en",
  "ignore_patterns": ["node_modules", "dist", ".git"],
  "formatter": {
    "biome": {
      "extensions": [".ts", ".tsx", ".js", ".jsx"]
    },
    "prettier": {
      "disabled": true
    }
  },
  "lsp": {
    "typescript": {
      "enabled": true
    },
    "oxlint": {
      "enabled": false
    }
  }
}
```
::

## Configuration Schema

| Property | Type | Description |
|----------|------|-------------|
| `language` | `"en"` \| `"ko"` | Language for messages |
| `ignore_patterns` | `string[]` | Patterns to ignore |
| `formatter` | `false` \| `object` | Formatter configuration |
| `lsp` | `false` \| `object` | LSP configuration |

## Disable All Features

### Disable all formatters

```yaml
formatter: false
```

### Disable all LSP

```yaml
lsp: false
```

## Native TypeScript 7 (typescript-go)

The `typescript` server automatically uses the native TypeScript 7 compiler
([typescript-go](https://devblogs.microsoft.com/typescript/announcing-typescript-7-0/))
when your project ships one â€” no configuration needed. At startup it inspects
the project's `node_modules` and, if it finds a native build, spawns
`tsc --lsp --stdio` (or `tsgo --lsp --stdio` from
[`@typescript/native-preview`](https://www.npmjs.com/package/@typescript/native-preview)).
Otherwise it falls back to the classic `typescript-language-server --stdio`.

A project is treated as native TypeScript 7 when any of these hold:

- `node_modules/typescript` is at major version 7 or higher
- the `typescript` package ships `lib/getExePath.js` (native launcher)
- the `typescript` package is missing `lib/tsserver.js` (classic entry point)
- a `node_modules/.bin/tsgo` binary is present (`@typescript/native-preview`)

Because it is one server that swaps its own binary, there is a single set of
diagnostics either way â€” you never get duplicates. To use it, just install a
native TypeScript 7 build in your project:

```bash
npm install --save-dev @typescript/native-preview
```

## Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `CODE_PROJECT_PATH` | Project root directory | cwd |
| `CLAUDE_PROJECT_DIR` | Set by Claude Code | - |
| `DORA_PROJECT_PATH` | Project root for Dora | cwd |
| `DORA_TIMEOUT` | Dora request timeout (ms) | 30000 |

## Config File Discovery

Code Please looks for configuration in this order:

1. `.please/config.yml`
2. `.please/config.json`
3. `dora.json` (legacy)
4. `opencode.json` (legacy)
