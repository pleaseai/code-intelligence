# LSP Configuration

Configure which language servers to use and how they behave.

## LSP Item Schema

Each LSP server can have these options:

```yaml
lsp:
  <server-name>:
    enabled: true           # Enable/disable this server
    root: ./backend         # Custom project root
    command: [cmd, args...] # Custom spawn command
```

## Built-in Servers

### TypeScript/JavaScript

```yaml
lsp:
  # Primary TypeScript server
  typescript:
    enabled: true

  # Additional linting with oxlint
  oxlint:
    enabled: true

  # Deno projects
  deno:
    enabled: true
```

### Python

```yaml
lsp:
  pyright:
    enabled: true
    root: ./backend  # If Python code is in subdirectory
```

### Go

```yaml
lsp:
  gopls:
    enabled: true
```

### Rust

```yaml
lsp:
  rust-analyzer:
    enabled: true
```

### Other Languages

```yaml
lsp:
  # Kotlin
  kotlin:
    enabled: true

  # Dart
  dart:
    enabled: true

  # Vue
  vue:
    enabled: true

  # Prisma
  prisma:
    enabled: true

  # ESLint
  eslint:
    enabled: true
```

## Custom Servers

Add your own LSP servers:

```yaml
lsp:
  my-lsp:
    command: [my-language-server, --stdio]
```

## Auto-Download Servers

Some servers are automatically downloaded when first used:

| Server | Source | Install Location |
|--------|--------|-----------------|
| Kotlin LSP | JetBrains | `~/.cache/dora/kotlin-lsp/` |
| Dart LSP | Dart SDK | `~/.cache/dora/dart-sdk/` |
| Vue LSP | npm | Project node_modules |
| Prisma LSP | npm | Project node_modules |
| ESLint LSP | GitHub | `~/.cache/dora/eslint-lsp/` |

## Disable Servers

### Disable all

```yaml
lsp: false
```

### Disable specific

```yaml
lsp:
  oxlint:
    enabled: false
```

## Custom Root Directory

For monorepos or when code is in a subdirectory:

```yaml
lsp:
  pyright:
    root: ./backend

  typescript:
    root: ./frontend
```

## Examples

### TypeScript only (no oxlint)

```yaml
lsp:
  oxlint:
    enabled: false
```

### Multiple project roots

```yaml
lsp:
  typescript:
    root: ./packages/web

  pyright:
    root: ./packages/api
```

### Disable Vue for non-Vue project

```yaml
lsp:
  vue:
    enabled: false
```

## Server Priority

When multiple servers support the same file type, they run in priority order. Higher priority servers are checked first.

## Troubleshooting

### Server not found

- Check that the server is installed globally or in node_modules
- For auto-download servers, ensure network access

### Wrong project root

- Set explicit `root` in configuration
- Check for lock files in expected location

### Slow startup

- First run may download/build servers
- Subsequent runs use cached servers
