# LSP Diagnostics

Code Please provides real-time type checking and error detection using Language Server Protocol (LSP). It supports 30+ language servers with automatic detection and setup.

## Supported Language Servers

| Language | Server | Root Detection |
|----------|--------|----------------|
| TypeScript/JavaScript | typescript-language-server | `package.json`, `bun.lock` |
| TypeScript/JavaScript | oxlint | `.oxlintrc.json`, `package.json` |
| Deno | deno lsp | `deno.json`, `deno.jsonc` |
| Python | pyright | `pyproject.toml`, `requirements.txt` |
| Go | gopls | `go.mod`, `go.work` |
| Rust | rust-analyzer | `Cargo.toml` |
| Kotlin | JetBrains Kotlin LSP | `build.gradle.kts`, `pom.xml` |
| Dart | dart language-server | `pubspec.yaml` |
| Vue | @vue/language-server | `package.json` (with vue) |
| Prisma | @prisma/language-server | `schema.prisma` |
| ESLint | vscode-eslint | `eslint.config.js`, `.eslintrc` |

## How It Works

1. When a file is checked, Code Please detects the language from the extension
2. It finds appropriate language servers and starts them if needed
3. The file is sent to the server for analysis
4. Diagnostics (errors, warnings) are collected and formatted
5. Results are returned to Claude

## Auto-Download Servers

Some language servers are automatically downloaded when needed:

- **Kotlin LSP** - Downloaded from JetBrains
- **Dart LSP** - Downloaded from Dart SDK
- **Vue LSP** - Installed via npm
- **Prisma LSP** - Installed via npm
- **ESLint LSP** - Built from vscode-eslint source

## Configuration

Create `.please/config.yml` in your project root:

```yaml
lsp:
  # Disable TypeScript server
  typescript:
    enabled: false

  # Custom root for Python
  pyright:
    root: ./backend

  # Custom command
  custom-lsp:
    command: [my-lsp-server, --stdio]
```

## Disable LSP

### Disable all LSP

```yaml
lsp: false
```

### Disable specific server

```yaml
lsp:
  oxlint:
    enabled: false
```

## Output Format

Errors are formatted for easy reading:

```
✗ 3 errors, 1 warning found
  ✗ index.ts:15:7 [TS2322]: Type 'string' is not assignable to type 'number'
  ✗ index.ts:20:5 [TS2304]: Cannot find name 'undefined_var'
  ✗ main.py:8:12: Undefined variable 'user'
  ⚠ utils.ts:25:10: Variable 'unused' is assigned but never used
```

## CLI Usage

```bash
# Check a specific file
code lsp src/index.ts

# Check via stdin (for hooks)
code lsp --stdin

# List available LSP servers
code lsp-server list
```

## LSP Operations

Beyond diagnostics, the LSP client supports:

- **Hover** - Get type information at a position
- **Definition** - Jump to definition
- **References** - Find all references
- **Completion** - Get code completions
- **Rename** - Rename symbols

These are available via the MCP server (Dora) for IDE integration.

## Troubleshooting

### Server not starting

- Check that the language server is installed
- Verify project root detection (check for lock files)
- Run `code lsp <file>` manually to see errors

### Slow diagnostics

- Some servers take time to initialize on first use
- Large projects may take longer to analyze
- Consider disabling servers you don't need
