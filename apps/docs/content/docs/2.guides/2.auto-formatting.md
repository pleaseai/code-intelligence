# Auto-Formatting

Code Please supports 20+ formatters out of the box. Files are automatically formatted using the appropriate formatter based on file extension.

## Supported Formatters

| Formatter | Extensions | Detection |
|-----------|------------|-----------|
| Biome | `.ts`, `.tsx`, `.js`, `.jsx`, `.json` | `biome.json` or installed |
| Prettier | `.ts`, `.tsx`, `.js`, `.jsx`, `.json`, `.md`, `.css` | `.prettierrc` or installed |
| gofmt | `.go` | Go installed |
| ruff | `.py` | `ruff.toml` or installed |
| rustfmt | `.rs` | Cargo project |
| clang-format | `.c`, `.cpp`, `.h` | `.clang-format` or installed |
| ktlint | `.kt`, `.kts` | Kotlin project |
| mix | `.ex`, `.exs` | Elixir project |
| dart | `.dart` | Dart SDK |
| zig fmt | `.zig` | Zig installed |
| ocamlformat | `.ml`, `.mli` | OCaml project |
| terraform | `.tf` | Terraform installed |
| gleam | `.gleam` | Gleam project |
| prisma | `.prisma` | Prisma project |

## How It Works

1. When a file is modified, Code Please detects the file extension
2. It finds the appropriate formatter based on extension and project config
3. The formatter is executed with the file path
4. The file is updated in place

## Configuration

Create `.please/config.yml` in your project root:

```yaml
formatter:
  # Override biome extensions
  biome:
    extensions: [.ts, .tsx, .js, .jsx, .json, .jsonc]

  # Disable prettier
  prettier:
    disabled: true

  # Add custom formatter
  custom-fmt:
    command: [my-formatter, format, $FILE]
    extensions: [.xyz]
```

## Disable Formatting

### Disable all formatters

```yaml
formatter: false
```

### Disable specific formatter

```yaml
formatter:
  prettier:
    disabled: true
```

## Custom Formatters

Add custom formatters in `.please/config.yml`:

```yaml
formatter:
  my-formatter:
    command: [my-tool, format, --write, $FILE]
    extensions: [.custom]
    environment:
      MY_VAR: value
```

- `command`: Array of command and arguments. Use `$FILE` as placeholder
- `extensions`: File extensions this formatter handles
- `environment`: Optional environment variables

## Formatter Priority

When multiple formatters support the same extension, priority is determined by:

1. Project-specific config files (e.g., `biome.json` for Biome)
2. Custom formatters defined in `.please/config.yml`
3. Built-in formatter order

## CLI Usage

```bash
# Format a specific file
code format src/index.ts

# Format via stdin (for hooks)
code format --stdin
```
